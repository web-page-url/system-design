# 1. What Codemia Actually Is (Product Breakdown)

Codemia is essentially **LeetCode for System Design**.

Instead of coding problems, it gives **system design challenges** like:

* Design URL Shortener
* Design Uber
* Design Dropbox
* Design Ticket Booking System

Users must:

1. Understand requirements
2. Estimate scale
3. Design architecture
4. Create diagrams
5. Submit solution
6. Get AI feedback

The platform includes:

* 120+ system design problems
* AI feedback system
* structured problem solving
* architecture diagrams
* editorial solutions
* progress tracking

The key idea is **interactive practice + AI evaluation**. ([javarevisited.wordpress.com][1])

---

### MASTER BUILD PROMPT

```
Build a production-grade web application similar to Codemia.io using Next.js 16.

The platform is an interactive system design practice platform where users solve real-world architecture problems and receive AI feedback.

Core Features

1 Problem Library
A list of system design problems similar to LeetCode.

Each problem contains:
- title
- description
- difficulty
- tags
- companies
- hints
- editorial solution

Users can filter problems by difficulty, tags, and company.

2 Interactive Solution Workspace

Users solve problems using structured sections:

Functional Requirements
Non Functional Requirements
Capacity Estimation
API Design
Database Design
High Level Architecture
Detailed Design
Tradeoffs
Future Improvements

3 Diagram Builder

Users should be able to draw architecture diagrams inside the workspace using a tool like tldraw or Excalidraw.

Diagrams must be saved as JSON in the database.

4 AI Evaluation

When the user submits a solution:

Send the full answer to an AI evaluation service.

AI should evaluate:

Requirements
Scalability
Architecture
Data modeling
Tradeoffs

Return a structured JSON score with feedback.

Store results in database.

5 Dashboard

User dashboard must show:

Problems solved
Average score
Recent submissions
Leaderboard

6 Editorial Solutions

Each problem should have expert solution after submission.

7 Authentication

Implement authentication using Clerk or NextAuth.

8 Database

Use PostgreSQL with Prisma ORM.

Tables:

Users
Problems
Solutions
Diagrams
EditorialSolutions

9 UI Design

Use:

Next.js 16
React Server Components
TailwindCSS
Shadcn UI
Framer Motion animations

Design should be modern and minimal similar to developer tools like:

Linear
Vercel
Stripe docs

10 Architecture

Use Next.js server actions for backend logic.

Integrate Redis caching.

AI evaluation should be handled asynchronously using a queue system.

11 Deployment

Deploy frontend on Vercel
Database on Supabase
Redis on Upstash

Ensure the application is mobile responsive, scalable, and production-ready.
```
